## Executive Summary

SIGYS (Sistema Integral de Guardias y Suplencias) es una iniciativa estratégica de modernización tecnológica impulsada para sustituir un sistema legacy con más de 20 años de operación dentro del ISSSTE.

La plataforma anterior representaba riesgos operativos, ausencia de validaciones normativas automatizadas y alta dependencia de intervención manual.  

SIGYS fue diseñado como un sistema nacional orientado a garantizar continuidad operativa en la atención médica, formalizando digitalmente reglas de operación, procesos de validación, generación de pagos y cumplimiento fiscal.

Este proyecto consolidó mi rol como arquitecto de solución con visión institucional y liderazgo técnico.

## Stack Tecnológico

- Spring Framework (Backend)
- Angular Framework (Frontend)
- API REST transaccional
- Informix
- Servidores Unix
- Integración con procesos de dispersión bancaria y timbrado fiscal

---

## Contexto Tecnológico y Organizacional

El sistema legacy presentaba:

- Más de 20 años en operación.
- Controles de seguridad limitados.
- Validaciones normativas no integradas.
- Dependencia significativa de procesos manuales.
- Riesgo operativo institucional.

El proceso que soporta es crítico, ya que garantiza la continuidad de atención médica ante ausentismo del personal de base mediante contratación de personal externo.

---

## Arquitectura y Decisiones Técnicas Clave

El modelo arquitectónico se diseñó bajo principios de:

- Separación clara entre frontend y backend.
- API REST centralizada.
- Control de acceso basado en perfiles.
- Validaciones normativas integradas en lógica de negocio.
- Trazabilidad completa de transacciones.

Participé directamente en:

- Diseño de modelos de dominio.
- Diseño de arquitectura general.
- Modelado de base de datos.
- Diseño e implementación inicial del API.
- Estrategias de seguridad y control de acceso.

Posteriormente delegué desarrollo operativo al equipo, enfocándome en dirección técnica y gestión.

---

## Modelo Operativo del Sistema

1. Apertura quincenal de captura en unidades médicas.
2. Registro de guardias y suplencias.
3. Validación por doble nivel de autoridad médica.
4. Autorización institucional.
5. Generación de pagos.
6. Dispersión bancaria.
7. Timbrado ante autoridades fiscales.

El sistema también administra padrón nacional de personal externo autorizado.
